Лабораторная работа 10 Инсталляция и запуск Oracle BI 11g в среде Microsoft Windows. Установка Oracle BI Administrative Tools.

1. Цель работы

Формирование практических умений по инсталляции Oracle BI 11g в среде Microsoft Windows, установке программного продукта Oracle BI Administrative Tools.

2. Учебный материал по лабораторной

Перед установкой OBIEE 11g необходимо обеспечить следующие условия:

Машина с установленной серверной версией Windows (32 или 64 bit) имеющая минимум 4Gb оперативной памяти, 20Gb свободного дискового пространства, 950Мб дискового пространства для временных файлов, 3Gb пространства под swap, dual core Pentium class процессор 1.5GHz или лучше.

Статический IP. Если используется DHCP, то потребуется установить Microsoft Loopback Adapter.

Установленный .NET Framework версии 1

Установленная СУБД Oracle (можно также DB2 или MSSQL).

Инсталляционный комплект Repository Creation Utility (11.1.1.6.0).

Инсталляционный комплект Business Intelligence 11g.

Все необходимые дистрибутивы можно скачать с http://edelivery.oracle.com/

Перед инсталляцией нужно выключить все антивирусные программы, иначе процесс может затянуться.

Доходим до шага Database Connecton Detail, вводим параметры соединения с базой данных репозитория (нужен пользователь с привилегией DBA или SYSDBA)

Далее, соглашаемся с созданием новых табличных пространств.

Далее проверяем информацию, введенную на предыдущих шагах, проверяем параметры и запускаем создание репозитория.

2. Инсталляция Oracle BI 11g

Запускаем ./runInstaller

Выбираем режим инсталляции Enterprise (можно и Simple).

Simple — простейший с минимумом вопросов

Enterprise — обычный режим

Software Only — проинсталлировать, но не конфигурировать BI

Ждем пока пройдут проверки на совместимость. Если возникнут проблемы, то исправляем и повторное запускаем проверки. Настройки подходят от базы данных. Жмем Next.

Указываем, что хотим создать новую систему, прописываем пароль и имя домена Weblogic, в который будет развернут BI:


  1. Указываем путь к домашнему каталогу Fusion Middleware (должен быть пустой каталог). После этого жмем Tab — все остальные поля будут заполнены автоматически. Имеет смысл запомнить значение Oracle Instance Location. Там будут лежать все конфигурационные файлы


2. Выбираем продукты, которые хотим проинсталлировать (их всего 3). Я оставил все.

3. Указываем подключение к базе данных в виде строки hostname:port:servicename. Указываем имя пользователя и пароль (мы его создали с помощью RCU, смотрите выше).

3. Запуск ORACLE BI 11G

После того, как Oracle BI проинсталлирован — у вас уже есть готовая, запущенная система. Давайте перезапустим компьютер. После этого, чтобы запустить BI нужно проделать следующие шаги:

В файлах

/u01/app/oracle/product/fmw/user_projects/domains/bifoundation_domain/servers/bi_server1/security/boot.properties
/u01/app/oracle/product/fmw/user_projects/domains/bifoundation_domain/servers/AdminServer/security/boot.properties

прописываем имя и пароль пользователя weblogic для упрощения жизни (после запуска пароли будут зашифрованы)

1. Запускаем базу с метаданными

2. Запускаем Weblogic Admin Server
/u01/app/oracle/product/fmw/user_projects/domains/bifoundation_domain/startWebLogic.sh

3. Запускаем Node Manager
/u01/app/oracle/product/fmw/wlserver_10.3/server/bin/startNodeManager.sh

4. Запускаем Weblogic сервер для BI
/u01/app/oracle/product/fmw/user_projects/domains/bifoundation_domain/bin/startManagedWebLogic.sh bi_server1

5. Запускаем OPMN. Он теперь отвечает за запуск всех процессов, который живут не внутри Weblogic
/u01/app/oracle/product/fmw/instances/instance1/bin/opmnctl start

6. Выполняем
/u01/app/oracle/product/fmw/instances/instance1/bin/opmnctl start
Ждем, пока все процессы будут Alive

http://server:9704/analytics

Для входа в BI используем имя weblogic и пароль welcome1.

Для управление используем http://server:7001/em

4. Загрузка и установка Oracle BI Administration Tool

Войдите в Oracle BI (http://server:9704/analytics), перейдите на главную страницу (нажать клавишу "Home" на панели инструментов), откройте меню "Скачать BI Desktop Tools" и выберите соответствующий (32 или 64 бит) установочный пакет . Во время процесса установки вы пройдете через создании ODBC подключения к BI Server. Соединение ODBC также могут быть созданы позже (как описано в разделе ниже).

5. Настройка ODBC подключения к Oracle BI Server

ODBC Data Source необходим для импорта информации о данных в репозитории Oracle BI Server.

Откройте Windows " Administration Tools" консоль и перейдите к " Data Sources (ODBC)”

Нажмите кнопку “Add” и выберите драйвер Oracle BI Server ODBC (обычно "Oracle BI Server, 1").

При настройке подключения наиболее важным полем здесь является "Server", где должен быть адрес BI сервера (имя хоста или IP-адрес).

Введите учетные данные для проверки конфигурации (данные для подключения к существующей базе данных (SH Schema))

Отрегулируйте параметры подключения (оставить их как они есть).

Нажмите кнопку "Finish". Подключение должно появиться в списке.

3. Порядок выполнения работы

1. Ознакомиться с учебным материалом

2. Проинсталлировать комплект Repository Creation Utility (11.1.1.6.0).

3. Проинсталлировать Oracle BI 11g

4. Запустить ORACLE BI 11G

5. Загрузить и установить Oracle BI Administration Tool

6. Создать и произвести настройку ODBC подключения к Oracle BI Server

7. Согласовать с преподавателем содержание отчёта и подготовить его.
